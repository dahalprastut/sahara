KNOWLEDGE BASE INGESTION PIPELINE - COMPLETE GUIDE
=====================================================

OVERVIEW:
The knowledge base is a collection of 27 documents (both handwritten and extracted 
from PDFs) that get loaded into memory when the server starts. When a user sends a 
query, the system searches these documents, retrieves the most relevant ones, and 
uses them to generate a personalized response via LLM.

STAGE 3: SEARCH/RETRIEVAL
================================================================================

When user sends query: "I have a job interview tomorrow and I'm nervous"

1. Query goes to RAGPipeline.generate_response()
2. Which calls retrieve_relevant_knowledge()
3. Which calls knowledge_base.search()

SEARCH ALGORITHM (Word Matching):

def search(query, category, top_k=3):
    query_lower = query.lower()
    results = []
    
    # Get documents in selected category
    category_docs = self.data.get(category, [])
    
    # Score each document
    for doc in category_docs:
        content = f"{doc['title']} {doc['content']}".lower()
        score = _similarity_score(query_lower, content)
        
        if score > 0:
            results.append({**doc, 'score': score})
    
    # Sort by score descending, return top-k
    results.sort(key=lambda x: x['score'], reverse=True)
    return results[:top_k]

SIMILARITY SCORING:

def _similarity_score(query, text):
    query_words = query.lower().split()
    text_lower = text.lower()
    
    # Count how many query words appear in text
    matches = sum(1 for word in query_words if word in text_lower)
    
    # Return percentage (0 to 1)
    return matches / len(query_words) if query_words else 0

EXAMPLE MATCHING:

Query: "I have a job interview tomorrow and I'm nervous"
Query words: ["i", "have", "a", "job", "interview", "tomorrow", "and", "i'm", "nervous"]

Document 1: "Job Interview Anxiety" (synthetic)
Content: "Job interviews trigger anxiety... Remember: the interviewer wants you to succeed..."
Matching words: ["job", "interview"] = 2 matches
Score: 2/9 = 0.22

Document 2: "Career Counseling strategies" (PDF)
Content: "Interview techniques and anxiety management..."
Matching words: ["interview"] = 1 match
Score: 1/9 = 0.11

Result: Document 1 has higher score, returns first

TOP-K RETRIEVAL (top_k=3):
Returns best 3 matching documents to augment the LLM prompt.

================================================================================
STAGE 4: AUGMENTATION (Building RAG Context)
================================================================================

File: app/services/rag_service.py

In generate_response():

1. Retrieve relevant knowledge:
   relevant_knowledge = self.retrieve_relevant_knowledge(
       user_query="I have a job interview...",
       category="careers",
       top_k=3
   )
   
   Returns: [
       {best matching doc with score: 0.75},
       {second best with score: 0.50},
       {third best with score: 0.45}
   ]

2. Get user context (for personalization):
   user_context = firebase_service.get_user_context(user_id)
   
   Returns: {
       "name": "Alex",
       "anxiety_type": "careers",
       "previous_triggers": ["Public speaking", "Job interviews"],
       "successful_coping": ["breathing exercises"]
   }

3. Build augmented prompt:

   prompt = f"""You are a compassionate mental health support specialist.
   
   Context Information:
   {json.dumps(relevant_knowledge[:2], indent=2)}
   
   User History:
   {user_context}
   
   User's anxiety type: {anxiety_type}
   User's Query: {user_query}
   
   Provide a supportive response that:
   1. Acknowledges feelings
   2. Provides practical advice based on context
   3. Is personalized to anxiety type
   (1-3 sentences)
   
   Response:"""

4. Send to LLM (Ollama llama3:8b):
   response = self.llm.invoke(prompt)

The RAG (Retrieval-Augmented Generation):
- RETRIEVAL: Gets relevant documents from knowledge base
- AUGMENTATION: Adds those documents to prompt as context
- GENERATION: LLM generates response informed by retrieved context

================================================================================
STAGE 5: RESPONSE & STORAGE
================================================================================

1. Sanitize response (safety checks):
   response_text = SafetyGuardrail.sanitize_response(response_text)

2. Add safety disclaimer:
   response_text += "\n\n⚠️ I'm an AI chatbot, not a licensed therapist..."

3. Store in Firebase:
   firebase_service.add_message(user_id, conv_id, {
       "role": "bot",
       "text": response_text,
       "responseType": "rag_response"
   })

4. Return to user:
   return ChatbotResponse(
       conversation_id=conv_id,
       response_text=response_text,
       timestamp=datetime.utcnow().isoformat()
   )

================================================================================
HOW TO ADD MORE DATA
================================================================================

METHOD 1: Add Hardcoded Synthetic Data

1. Open: app/knowledge_base/synthetic_data.py
2. Find the _initialize_data() method
3. Add new dict to appropriate list:

   {
       "id": "career_6",
       "title": "My New Topic",
       "content": "Full text here...",
       "category": "careers",
       "type": "advice"
   }

4. Restart server
5. New document automatically in knowledge base

METHOD 2: Add PDFs (Automatic)

1. Drop PDF files into: documents/ folder
2. Run: python extract_pdfs.py
3. Wait for completion
4. Restart server: python main.py
5. PDFs automatically parsed, categorized, and loaded!

METHOD 3: Connect External Database (Advanced)

Modify _initialize_data() to query your own database:

def _initialize_data(self):
    data = {...}  # existing synthetic data
    
    # Query your database
    db_docs = my_database.query_all_documents()
    for doc in db_docs:
        category = doc.get_category()
        data[category].append({
            "id": doc.id,
            "title": doc.title,
            "content": doc.content,
            "category": category,
            "source": "database"
        })
    
    return data

================================================================================
SUMMARY
================================================================================

The knowledge base ingestion pipeline is a 5-stage process:

1. DATA SOURCES: Synthetic data + PDFs
2. LOADING: Combined into one in-memory dictionary
3. RETRIEVAL: Word-matching search returns top-k relevant docs
4. AUGMENTATION: Retrieved docs injected into LLM prompt
5. RESPONSE: LLM generates informed response, safety checked

The result: Personalized, counseling-informed responses that cite
professional sources and provide actionable advice.

Total documents: 27 (easily expandable)
Update frequency: Restart server to pick up new PDFs
Customization: Fully modular, can upgrade each component
